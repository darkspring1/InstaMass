Требования.

Иметь возможность настроить расписание активность для каждого аккаунта
Возможность отключать/включать активности (лайки, подписки)
Для одного аккаунта действия должны выполнятся последовательно, через случайные промежуток времени, как бы это делал реальный человек.
Для одного аккаунта количество действий в час не должно привышать лимитов в час.

Алгоритм работы.
для каждого пользователя создаётся расписание на сутки

Виды действий:
like по хеш-тегу
like по месту
подписка
отписка
... ещё что то

логи/пароль | время (чч.мм.сс) | действие    unique_id
super/vasia		1:04:30		     like tag



один актор периодический проверяет расписание
actions =   select * from SCHEDULER where TIME < CURRENT_TIME


Группируем actions по пользователю

Для каждой группы создаём ActionActor.
Актор должен выполнять действия последовательно, как бы это делал настоящий человек.
Следовательно для каждого пользователя в системе, одномоментно может сущестоввать только единственный такой актор.
После выполнения, каждого действия ActionActor сигнализирует родителю, о выполнения действия.

Устройство ActionActor:
PersistentActor
PersistentID: action_date + user_login
State: список заданий, выполненные и вновь поступившие
